Return the default device ID or pm\+No\+Device if there are no devices. The result (but not pm\+No\+Device) can be passed to Pm\+\_\+\+Open\+Midi().

The default device can be specified using a small application named pmdefaults that is part of the Port\+Midi distribution. This program in turn uses the Java Preferences object created by java.\+util.\+prefs.\+Preferences.\+user\+Root().node(\char`\"{}/\+Port\+Midi\char`\"{}); the preference is set by calling prefs.\+put(\char`\"{}\+P\+M\+\_\+\+R\+E\+C\+O\+M\+M\+E\+N\+D\+E\+D\+\_\+\+O\+U\+T\+P\+U\+T\+\_\+\+D\+E\+V\+I\+C\+E\char`\"{}, pref\+Name); or prefs.\+put(\char`\"{}\+P\+M\+\_\+\+R\+E\+C\+O\+M\+M\+E\+N\+D\+E\+D\+\_\+\+I\+N\+P\+U\+T\+\_\+\+D\+E\+V\+I\+C\+E\char`\"{}, pref\+Name);

In the statements above, pref\+Name is a string describing the M\+I\+DI device in the form \char`\"{}interf, name\char`\"{} where interf identifies the underlying software system or A\+PI used by Port\+Mdi to access devices and name is the name of the device. These correspond to the interf and name fields of a \hyperlink{struct_pm_device_info}{Pm\+Device\+Info}. (Currently supported interfaces are \char`\"{}\+M\+M\+System\char`\"{} for Win32, \char`\"{}\+A\+L\+S\+A\char`\"{} for Linux, and \char`\"{}\+Core\+M\+I\+D\+I\char`\"{} for OS X, so in fact, there is no choice of interface.) In \char`\"{}interf, name\char`\"{}, the strings are actually substrings of the full interface and name strings. For example, the preference \char`\"{}\+Core, Sport\char`\"{} will match a device with interface \char`\"{}\+Core\+M\+I\+D\+I\char`\"{} and name \char`\"{}\+In U\+S\+B Midi\+Sport 1x1\char`\"{}. It will also match \char`\"{}\+Core\+M\+I\+D\+I\char`\"{} and \char`\"{}\+In U\+S\+B Midi\+Sport 2x2\char`\"{}. The devices are enumerated in device ID order, so the lowest device ID that matches the pattern becomes the default device. Finally, if the comma-\/space (\char`\"{}, \char`\"{}) separator between interface and name parts of the preference is not found, the entire preference string is interpreted as a name, and the interface part is the empty string, which matches anything.

On the M\+AC, preferences are stored in /\+Users/\$\+N\+A\+ME/\+Library/\+Preferences/com.apple.\+java.\+util.\+prefs.\+plist which is a binary file. In addition to the pmdefaults program, there are utilities that can read and edit this preference file.

Normally, the client finds out about errors through returned error codes, but some errors can occur asynchronously where the client does not explicitly call a function, and therefore cannot receive an error code. The client can test for a pending error using \hyperlink{portmidi_8h_aa631918a228994f19bed5dcbc398d59a}{Pm\+\_\+\+Has\+Host\+Error()}. If true, the error can be accessed and cleared by calling \hyperlink{portmidi_8h_af6f9fa86b2192d5df87a36762f43b1a4}{Pm\+\_\+\+Get\+Error\+Text()}. Errors are also cleared by calling other functions that can return errors, e.\+g. \hyperlink{group__grp__device_gabd50a31baaa494ad8b405f9ad54c966e}{Pm\+\_\+\+Open\+Input()}, \hyperlink{group__grp__device_ga134924cfa8badeecff3c5e1f22aee178}{Pm\+\_\+\+Open\+Output()}, \hyperlink{group__grp__io_ga3d59225bc890ede974f245ada3de6456}{Pm\+\_\+\+Read()}, \hyperlink{group__grp__io_ga82950117f003d28f9ca6536c00af985a}{Pm\+\_\+\+Write()}.
