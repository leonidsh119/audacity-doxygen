\subsection{Detailed Description}
If you want to compile using F\+F\+TW instead of the built-\/in \hyperlink{class_vamp_1_1_f_f_t}{F\+FT} implementation for the Plugin\+Input\+Domain\+Adapter, define H\+A\+V\+E\+\_\+\+F\+F\+T\+W3 in the Makefile.

Be aware that F\+F\+TW is licensed under the G\+PL -- unlike this S\+DK, which is provided under a more liberal B\+SD license in order to permit use in closed source applications. The use of F\+F\+TW would mean that your code would need to be licensed under the G\+PL as well. Do not define this symbol unless you understand and accept the implications of this.

Parties such as Linux distribution packagers who redistribute this S\+DK for use in other programs should {\itshape not} define this symbol, as it would change the effective licensing terms under which the S\+DK was available to third party developers.

The default is not to use F\+F\+TW, and to use the built-\/in \hyperlink{class_vamp_1_1_f_f_t}{F\+FT} instead.

Note\+: The F\+F\+TW code uses F\+F\+T\+W\+\_\+\+M\+E\+A\+S\+U\+RE, and so will perform badly on its first invocation unless the host has saved and restored F\+F\+TW wisdom (see the F\+F\+TW documentation).
