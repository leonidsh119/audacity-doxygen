Prototype for plugin accessor function.

Plugins are discovered by hosts using R\+DF data (not by loading libraries). See \href{http://lv2plug.in}{\tt http\+://lv2plug.\+in} for details on the discovery process, though most hosts should use an existing library to implement this functionality.

This is the simple plugin discovery A\+PI, suitable for most statically defined plugins. Advanced plugins that need access to their bundle during discovery can use \hyperlink{lv2_8h_a9eb5804c670849d3faafed9f9cd65f23}{lv2\+\_\+lib\+\_\+descriptor()} instead. Plugin libraries M\+U\+ST include a function called \char`\"{}lv2\+\_\+descriptor\char`\"{} or \char`\"{}lv2\+\_\+lib\+\_\+descriptor\char`\"{} with C-\/style linkage, but S\+H\+O\+U\+LD provide \char`\"{}lv2\+\_\+descriptor\char`\"{} wherever possible.

When it is time to load a plugin (designated by its U\+RI), the host loads the plugin\textquotesingle{}s library, gets the \hyperlink{amp_8c_affedc7aabe2715522365f21a9c1ff65d}{lv2\+\_\+descriptor()} function from it, and uses this function to find the L\+V2\+\_\+\+Descriptor for the desired plugin. Plugins are accessed by index using values from 0 upwards. This function M\+U\+ST return N\+U\+LL for out of range indices, so the host can enumerate plugins by increasing {\ttfamily index} until N\+U\+LL is returned.

Note that {\ttfamily index} has no meaning, hosts M\+U\+ST N\+OT depend on it remaining consistent between loads of the plugin library.

The \hyperlink{amp_8c_affedc7aabe2715522365f21a9c1ff65d}{lv2\+\_\+descriptor()} function is the entry point to the plugin library. The host will load the library and call this function repeatedly with increasing indices to find all the plugins defined in the library. The index is not an indentifier, the U\+RI of the returned descriptor is used to determine the identify of the plugin.

Prototype for library accessor function.

This is the more advanced discovery A\+PI, which allows plugin libraries to access their bundles during discovery, which makes it possible for plugins to be dynamically defined by files in their bundle. This A\+PI also has an explicit cleanup function, removing any need for non-\/portable shared library destructors. Simple plugins that do not require these features may use \hyperlink{lv2_8h_affedc7aabe2715522365f21a9c1ff65d}{lv2\+\_\+descriptor()} instead.

This is the entry point for a plugin library. Hosts load this symbol from the library and call this function to obtain a library descriptor which can be used to access all the contained plugins. The returned object must not be destroyed (using \hyperlink{struct_l_v2___lib___descriptor_af74270275ff1c5931ef2ab8a57f77065}{L\+V2\+\_\+\+Lib\+\_\+\+Descriptor\+::cleanup()}) until all plugins loaded from that library have been destroyed.
