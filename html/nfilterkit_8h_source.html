28 /* Description of constants:
29  *
30  * Npc - is the number of look-up values available for the lowpass filter
31  * between the beginning of its impulse response and the "cutoff time"
32  * of the filter. The cutoff time is defined as the reciprocal of the
33  * lowpass-filter cut off frequence in Hz. For example, if the
34  * lowpass filter were a sinc function, Npc would be the index of the
35  * impulse-response lookup-table corresponding to the first zero-
36  * crossing of the sinc function. (The inverse first zero-crossing
37  * time of a sinc function equals its nominal cutoff frequency in Hz.)
38  * Npc must be a power of 2 due to the details of the current
39  * implementation. The default value of 512 is sufficiently high that
40  * using linear interpolation to fill in between the table entries
41  * gives approximately 16-bit accuracy in filter coefficients.
42  *
43  * Nhc - is log base 2 of Npc.
44  *
45  * Na - is the number of bits devoted to linear interpolation of the
46  * filter coefficients.
47  *
48  * Np - is Na + Nhc, the number of bits to the right of the binary point
49  * in the integer "time" variable. To the left of the point, it indexes
50  * the input array (X), and to the right, it is interpreted as a number
51  * between 0 and 1 sample of the input X. Np must be less than 16 in
52  * this implementation.
53  *
54  * Nh - is the number of bits in the filter coefficients. The sum of Nh and
55  * the number of bits in the input data (typically 16) cannot exceed 32.
56  * Thus Nh should be 16. The largest filter coefficient should nearly
57  * fill 16 bits (32767).
58  *
59  * Nb - is the number of bits in the input data. The sum of Nb and Nh cannot
60  * exceed 32.
61  *
62  * Nhxn - is the number of bits to right shift after multiplying each input
63  * sample times a filter coefficient. It can be as great as Nh and as
64  * small as 0. Nhxn = Nh-2 gives 2 guard bits in the multiply-add
65  * accumulation. If Nhxn=0, the accumulation will soon overflow 32 bits.
66  *
67  * Nhg - is the number of guard bits in mpy-add accumulation (equal to Nh-Nhxn)
68  *
69  * NLpScl - is the number of bits allocated to the unity-gain normalization
70  * factor. The output of the lowpass filter is multiplied by LpScl and
71  * then right-shifted NLpScl bits. To avoid overflow, we must have
72  * Nb+Nhg+NLpScl < 32.
73  */
