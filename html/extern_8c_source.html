3 /* Description of external types:
4 
5  A new node type EXTERN has been created to allow extensions to the
6 xlisp type system. If a node is of type EXTERN then it contains two fields:
7 (1) a pointer to a type descriptor, and (2) a value which is normally a pointer
8 to an instance of the data type.
9  The type descriptor has 6 fields:
10 type_name: a string name for the type
11 type_symbol: a pointer to a symbol whose print name is type_name
12 free_meth: routine to call when the node is freed
13 print_meth: routine to call to print the node
14 save_meth: routine to save the value to a file
15 restore_meth: routine to restore the value from a file
16  The routine create_desc() is used to allocate and initialize a type
17 descriptor. There should be only one type descriptor for each unique type.
18 Typically, the module that manages a type will create the type descriptor
19 at initialization time.
20  The routine cvextern() is used to combine a type descriptor and a
21 value to create an EXTERN node that can be used by xlisp. This routine is
22 called whenever a new value is allocated and returned to xlisp.
23  The routine exttype() returns the type symbol for an EXTERN type.
24  The routine exttypep() tests for a type match.
25  If EXTERN objects are dynamically allocated and freed, then there
26 should only be one EXTERN node whose value field (xe_inst) points to the object.
27 If there is more than one node (normally created only by cvextern), then one
28 of the following should hold:
29 1. the object is not dynamically freed.
30 2. the object is reference counted and freed when the last EXTERN node is gc'd.
31 3. the object will be freed when the first node pointing to it is gc'd, resulting
32 in a dangling pointer bug.
33  The save and restore capability of xlisp version 2.0 causes some
34 difficulties in that symbols get relocated whenever a workspace is loaded.
35 However, type descriptors must point to symbols and type descriptors must be
36 present in order to load external types from a workspace. My solution to this
37 problem is to store a string name for each type and also to cache a pointer
38 to the corresponding symbol. The cache is invalidated whenever a workspace is
39 loaded (causing symbols to be reallocated). To make it possible to save and
40 restore references to type descriptors, all type descriptors are kept in
41 static storage. Internally, they are referenced by pointers, but when saved,
42 an integer index is used. Because of the integer index, types must always be
43 allocated in the same order. This is taken care of in localinit().
44 
45 */
