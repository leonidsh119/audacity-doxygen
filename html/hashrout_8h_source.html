10 /*
11  * Generic symbol table functions
12  *
13  * After writing this code over for a bunch of interpreters
14  * I think I know how to do it once and for all, generally
15  * enough for most application.
16  * There are enough settable parameters to suit anyone, and
17  * the defaults usually do something sane.
18  *
19  * The basic idea is that you have a bunch of symbol table entries
20  * that need to be entered or looked up given a character string.
21  * Symbol table entries are usually structures, and one element
22  * of the structure must be the character string of the key.
23  * The structure should be given a type name, and these routines
24  * are informed of the type name via
25  * #define HASHTYPE type-name-of-symbol-table-entry
26  * You must inform the routines how to access the character string
27  * given the symbol table entry; for example
28  * #define HASHELEM(p) ((p).name)
29  * There are two size parameters - the number of different hash
30  * values and the number of symbol table entries to allocate at one
31  * time. Both default to 256.
32  * #define HASHVAL 128
33  * #define HASHENTRIES 512
34  * The name of the function that performs both entry and lookup is
35  * enter(name) - it takes one argument, a character string. The name
36  * of the function may be changed via
37  * #define HASHENTER new-name-for-enter-routine
38  * Note that if there is no entry in the table for name, name is entered
39  * automatically. The returned structure will have only the name
40  * field filled in; the rest of the fields are guarenteed to be zero
41  * so an application can check if this is the first time name was entered.
42  * If HASHPOINT is defined, the enter routine returns a pointer to a hash
43  * table entry. In the default case, i.e. HASHPOINT undefined, the
44  * enter routine returns an integer between zero and HASHENTRIES.
45  * The macro HASHENTRY(i) will, given that integer, return the
46  * table element (not a pointer to it), so for example,
47  * HASHENTRY(enter(x)).name == x.
48  * Any file that wishes to use HASHENTRY should have at the top
49  * #define HASHTYPE type-name-of-symbol-table-entry
50  * #include "hash.h"
51  * Note in this case at most HASHENTRIES entries will be allocated
52  * before hash table overflow. If HASHPOINT is defined, allocations
53  * will take place until memory runs out.
54  * By default, hash strings are copied into space obtained from malloc
55  * before being placed in new entry. This copying can be supressed
56  * by defining HASHNOCOPY.
57  * The following is an example of using the hash table stuff.
58
