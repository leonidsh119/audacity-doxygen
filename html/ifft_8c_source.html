31 /* index: index into outbuf whree we get output samples
32  * length: size of the frame, window, and outbuf; half size of samples
33  * array: spectral frame goes here (why not a local var?)
34  * window_len: size of window, should equal length
35  * outbuf: real part of samples are multiplied by window and added to
36  * outbuf (after shifting)
37  * src: send :NEXT to this object to get next frame
38  * stepsize: shift by this many and add each frame
39  * samples: result of ifft goes here, real and imag
40  * window: multiply samples by window if any
41  *
42  * IMPLEMENTATION NOTE:
43  * The src argument is an XLisp object that returns either an
44  * array of samples or NIL. The output of ifft is simply the
45  * concatenation of the samples taken from the array. Later,
46  * an ifft will be plugged in and this will return overlapped
47  * adds of the ifft's.
48  *
49  * OVERLAP: stepsize must be less than or equal to the length
50  * of real part of the transformed spectrum. A transform step
51  * works like this:
52  * (1) shift the output buffer by stepsize samples, filling
53  * the end of the buffer with zeros
54  * (2) get and transform an array of spectral coefficients
55  * (3) multiply the result by a window
56  * (4) add the result to the output buffer
57  * (5) output the first stepsize samples of the buffer
58  *
59  * DATA FORMAT: the DC component goes in array elem 0
60  * Cosine part is in elements 2*i-1
61  * Sine part is in elements 2*i
62  * Nyquist frequency is in element length-1
63  */
